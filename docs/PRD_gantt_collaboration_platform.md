# PRD: 부서 협업 간트차트 자동화 플랫폼

## 1. 서비스 목적 및 배경

### 1.1 목적
다부서 협업 환경에서 프로젝트와 할일을 실시간으로 공유하고, 진행상황을 시각적으로 추적할 수 있는 간트차트 기반 협업 플랫폼 개발

### 1.2 배경
- 기존 이메일/메신저 기반 업무 소통의 비효율성
- 부서별 업무 진행상황 가시성 부족
- 프로젝트 일정 지연 및 병목 구간 파악 어려움
- 엑셀 기반 간트차트의 실시간 동기화 한계
- 원격/하이브리드 근무 환경에서 협업 도구 필요성 증가

## 2. 문제 정의

### 2.1 현재 Pain Points
- **가시성 부족**: 타 부서 업무 진행상황을 실시간으로 파악 불가
- **중복 작업**: 동일한 정보를 여러 채널에 반복 공유
- **일정 관리 혼란**: 엑셀 파일 버전 관리 문제, 최신 정보 불일치
- **커뮤니케이션 분산**: 업무 관련 대화가 여러 도구에 분산
- **진행률 추적 어려움**: 수동 업데이트로 인한 정확도 저하

### 2.2 타겟 사용자
- **1차**: 중소기업(10~100명) 프로젝트 관리자, 팀장
- **2차**: 일반 팀원, 부서원
- **3차**: 경영진 (대시보드 모니터링)

## 3. 사용자 시나리오

### 시나리오 1: 프로젝트 매니저 (김팀장)
```
상황: 신제품 출시 프로젝트 총괄
행동:
1. 프로젝트 생성 및 주요 마일스톤 설정
2. 부서별 담당자 할당 (개발팀, 디자인팀, 마케팅팀)
3. 각 부서 하위 태스크 생성 및 의존성 설정
4. 간트차트에서 전체 일정 시각화
5. 지연 태스크 자동 알림 수신
6. 주간 진행률 리포트 확인

기대: 전체 프로젝트 진행상황을 한눈에 파악, 병목 구간 즉시 식별
```

### 시나리오 2: 개발팀 실무자 (박개발)
```
상황: 신제품 백엔드 API 개발 담당
행동:
1. 할당된 태스크 목록 확인
2. 태스크 시작 시 상태 '진행중'으로 변경
3. 작업 중 이슈 발생 시 코멘트로 타 부서와 소통
4. 완료 후 진행률 100% 업데이트
5. 후속 태스크 담당자에게 자동 알림 전송

기대: 자신의 업무에 집중하면서 전후 프로세스 파악 가능
```

### 시나리오 3: 경영진 (대표이사)
```
상황: 회사 전체 프로젝트 현황 모니터링
행동:
1. 대시보드 접속
2. 부서별/프로젝트별 진행률 한눈에 확인
3. 지연 프로젝트 상세 정보 드릴다운
4. 월간 완료율 트렌드 분석

기대: 데이터 기반 의사결정, 리소스 재배치 판단 근거 확보
```

## 4. 핵심 기능 (MVP 기준)

### 4.1 필수 기능 (Phase 1)
1. **사용자 인증**
   - 이메일/비밀번호 회원가입 및 로그인
   - 조직 생성 및 멤버 초대 (이메일 링크)
   - 권한 관리: Admin(관리자), Member(일반), Viewer(읽기전용)

2. **프로젝트 관리**
   - 프로젝트 CRUD
   - 부서/팀 단위 그룹핑
   - 프로젝트 상태: 계획중, 진행중, 완료, 보류
   - 색상 라벨링 (커스텀 색상 지정)

3. **태스크 관리**
   - 태스크 CRUD (제목, 설명, 담당자, 시작일, 마감일, 우선순위)
   - 하위 태스크 생성 (최대 3단계)
   - 태스크 의존성 설정 (선행 태스크 완료 후 시작 가능)
   - 진행률 수동 입력 (0~100%)
   - 상태 관리: 대기, 진행중, 완료, 지연

4. **간트차트 시각화**
   - 타임라인 뷰 (일/주/월 단위 전환)
   - 드래그&드롭으로 일정 조정
   - 태스크 간 의존성 화살표 표시
   - 오늘 날짜 하이라이트 (Today Line)
   - 부서별 색상 구분
   - 임계 경로(Critical Path) 자동 표시

5. **실시간 협업**
   - WebSocket 기반 동시 접속 (최대 50명)
   - 태스크별 댓글/멘션 기능
   - 변경사항 실시간 반영 (누가 어떤 작업을 수정했는지 표시)
   - 활동 로그 (Activity Feed)

6. **알림 시스템**
   - 태스크 할당 시 이메일 알림
   - 마감일 임박 알림 (D-3, D-1, 당일)
   - 멘션 시 인앱 알림
   - 태스크 완료 시 후속 담당자 자동 알림

7. **대시보드**
   - 전체 프로젝트 진행률 (파이차트)
   - 부서별 업무 부하 (Bar Chart)
   - 이번 주 마감 태스크 목록
   - 지연 태스크 하이라이트

### 4.2 확장 기능 (Phase 2)
- 캘린더 뷰 (월간 일정 표시)
- 칸반 보드 뷰 (드래그&드롭)
- 파일 첨부 (태스크별 최대 10MB)
- 반복 태스크 설정 (매주/매월)
- 템플릿 기능 (자주 사용하는 프로젝트 구조 저장)
- Excel/PDF 간트차트 내보내기
- Slack/카카오워크 연동

## 5. 기술적 요구사항

### 5.1 성능 기준
- 페이지 로드 시간: 3초 이내
- 간트차트 렌더링: 100개 태스크 기준 2초 이내
- 실시간 동기화 지연: 1초 이내
- 동시 접속: 조직당 최대 50명

### 5.2 브라우저 지원
- Chrome 최신 버전
- Safari 최신 버전
- Edge 최신 버전
- 모바일 반응형 (태블릿 이상 권장)

### 5.3 보안 요구사항
- HTTPS 필수
- JWT 기반 인증
- XSS/CSRF 방어
- 비밀번호 암호화 (bcrypt)
- 조직 간 데이터 격리 (RLS)

## 6. UI/UX 원칙

### 6.1 디자인 철학
- **미니멀리즘**: Monday.com 스타일의 깔끔한 인터페이스
- **직관성**: 첫 사용자도 5분 안에 태스크 생성 가능
- **반응성**: 모든 액션에 즉각적인 피드백
- **일관성**: 통일된 컬러 팔레트 및 컴포넌트

### 6.2 핵심 화면 구성
1. **로그인/회원가입**: 심플한 폼 (소셜 로그인 없음)
2. **대시보드**: 프로젝트 카드 그리드 뷰
3. **간트차트**: 전체 화면 활용, 좌측 태스크 리스트 + 우측 타임라인
4. **태스크 상세**: 우측 사이드 패널 (모달 아님)
5. **설정**: 조직 정보, 멤버 관리, 프로필

### 6.3 컬러 스킴
- Primary: #5B4FFF (보라)
- Success: #00C875 (초록)
- Warning: #FDAB3D (주황)
- Danger: #E2445C (빨강)
- Background: #F6F7FB (밝은 회색)
- Text: #323338 (다크 그레이)

## 8. 성공 지표 (KPI)

### 8.1 사용자 지표
- MAU (월간 활성 사용자): 100명 (6개월 내)
- 사용자 리텐션: 60% (30일 기준)
- 평균 세션 시간: 15분 이상

### 8.2 기능 지표
- 프로젝트 생성률: 사용자당 평균 2개
- 태스크 완료율: 70% 이상
- 댓글 활성도: 태스크당 평균 3개 이상

### 8.3 기술 지표
- 시스템 가동률: 99.5% 이상
- API 응답 시간: 평균 200ms 이내
- 에러율: 1% 미만
